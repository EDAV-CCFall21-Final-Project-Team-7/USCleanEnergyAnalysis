3.1.1 Correcting Date format

In the dataset for oil data there were different types of date formats such as ‘MM/DD/YY’ and ‘MM/DD/YYYY’ as shown below.

## # A tibble: 10 × 2
##    Day        `Cost per barrel`
##    <chr>                  <dbl>
##  1 11/08/21                82.0
##  2 11/05/21                81.2
##  3 11/04/21                78.9
##  4 11/03/21                80.8
##  5 11/02/21                83.9
##  6 11/01/21                84.1
##  7 10/29/2021              83.5
##  8 10/28/2021              82.8
##  9 10/27/2021              82.7
## 10 10/26/2021              85.6

We updated the dataset to contain dates in only one format (“MM/DD/YYYY”). This was done by splitting the date based on “/” as the delimiter.

##      [,1] [,2] [,3]
## [1,] "11" "08" "21"

We then checked if the date was already in the correct format by checking the number of characters in the year component of the split. If it was not correct then we added the missing components to the year format and joined the string again while updating that in the dataframe. If the year component in the was an integer less than 22 then we added “20” before the date making the year “20xx” but if the integer was greater than 22, it is obvious that it cannot be a date from the 21th century therefore we added “19” before the date making it “19xx”.

3.3 EIA Data

3.3.1 Corrections

There were several issues with the EIA data. The clean version of the data can be found here. First, the state name read in as “USA-” (examples below).

## [1] "USA-AL" "USA-AK" "USA-AZ" "USA-AR" "USA-CA"

We updated this column by splitting on the “-” delimiter and only keeping the actual state name.

## [1] "AL" "AK" "AZ" "AR" "CA"

Other formatting issues existed with the fuel type. There was white space at the start and end of each fuel and some fuels had a space in the middle of it (i.e. natural gas ) which is shown below.

## [1] " coal "              " petroleum liquids " " petroleum coke "   
## [4] " natural gas "       " other gases "

To make these variables easier to work with, we trimmed the white space at the beginning and end of the fuels and added an underscore if there was a space in the middle of the fuel type. A sample of the updated fuels is shown below:

## [1] "coal"              "petroleum_liquids" "petroleum_coke"   
## [4] "natural_gas"       "other_gases"

The “date” column of the data was imported as a character column, so we updated it to be a date datatype.

Lastly, there was an issue where the solar generation total was not aggregated correctly on the EIA website. This is detailed more in the missing values chapter. To fix this, we performed a pivot on the data and recalculated the aggregation, ignoring N/a’s in the data. We also imputed zero for other values of the power generation so that aggregations were not affected in later code.

3.3.2 Create interactive data

The data needed to be transformed for the interactive component of this project. The data is wrangled to create dataframes (listed as d2001.csv - d2021.csv) that contain the renewable percent of total generation for each state for each year as shown below. As mentioned in the data introduction, the total power generation is the aggregate of coal, petroleum_liquids, petroleum_coke, natural_gas, other_gases, nuclear, conventional_hydroelectric, other, and other_renewables_(total) fuel types. Additionally, the renewables value is the sum of the conventional_hydroelectric and other_renewables_(total) fuels. After performing these aggregations, the percent of renewables was found by dividing the aggregate renewables by the total power generation for each state on each date.
